# 界面设计

## 设计目标

本软件是一个同一程序支持多种角色的组网工具。界面设计需要同时服务三类用户：服务端管理员、客户现场普通客户端用户、实施人员。界面不只展示连接状态，还要把“节点、网段、映射地址、访问权限、路由状态”这些容易产生误解的概念用清晰方式呈现出来。

首发版本界面覆盖：

```text
1. 服务端 Web 后台
2. Windows 桌面端
3. Linux 桌面端
```

后续版本界面覆盖：

```text
1. Android 客户端模式
2. Android 实施端模式
3. Windows 服务端模式
```

本文件用于约束后续产品和开发实现，避免不同平台界面各做各的，导致功能概念不一致。

## 统一设计原则

### 角色优先

软件启动后先明确当前运行角色：

```text
服务端
普通客户端
实施端
```

不同角色看到的界面、操作入口和权限范围不同。服务端强调管理，普通客户端强调网段上报和连接状态，实施端强调可访问网段和远端访问诊断。

### 映射关系必须可见

用户访问远端设备时使用映射 IP，不直接使用客户现场真实 IP。界面必须始终展示真实网段和映射网段的对应关系。

示例展示方式：

```text
客户 A / 生产网
真实网段：192.168.1.0/24
映射网段：100.64.1.0/24
访问示例：192.168.1.10 -> 100.64.1.10
```

### 状态表达一致

所有平台统一使用以下状态：

| 状态 | 含义 |
|---|---|
| 未配置 | 尚未填写服务端地址或必要参数 |
| 未连接 | 已配置但未连接服务端 |
| 连接中 | 正在尝试连接服务端 |
| 已连接 | 已连接服务端，基础通信正常 |
| 已组网 | 虚拟网卡、路由和隧道均已就绪 |
| 部分异常 | 已连接但部分网段、路由或转发异常 |
| 已断开 | 与服务端连接中断 |

界面中不要只显示“在线/离线”。组网软件需要区分“服务端连上了”和“路由真正生效了”。

### 操作可回退

用户添加的路由、虚拟网卡状态、上报网段、权限配置都需要能撤销或停用。尤其是客户端和实施端退出软件时，要提示是否清理本机路由。

### 首发不做复杂视觉

首发界面优先清晰、稳定、可诊断。视觉风格建议简洁工程化，不追求大量动画和装饰。

## 产品信息架构

整体界面分为三套入口：

```text
服务端 Web 后台
桌面端客户端/实施端界面
Android 移动端界面
```

其中桌面端启动后根据角色进入不同页面：

```text
启动页
  ├─ 服务端模式
  ├─ 普通客户端模式
  └─ 实施端模式
```

服务端模式首发主要通过 Web 后台管理。桌面端服务端模式可以先只提供“启动服务、打开后台、查看状态”的轻量入口。

## 启动页设计

### 页面目标

启动页用于选择软件运行角色。它是所有桌面平台的统一入口。

### 页面结构

```text
软件名称
当前版本

请选择运行模式

[服务端模式]
用于部署中心服务，管理节点、网段和权限

[普通客户端模式]
用于客户现场电脑，上报本地可访问网段

[实施端模式]
用于实施人员电脑，访问已授权远端网段

底部：
设置
日志目录
帮助
```

### 交互规则

```text
1. 首次启动必须选择角色。
2. 角色选择后可以记住上次选择。
3. 用户可以在设置中切换默认角色。
4. 若切换角色，需要先停止当前连接或服务。
5. 服务端模式需要提示建议使用 Linux/Docker 部署。
```

### 网段管理

#### 页面目标

管理普通客户端上报的真实网段。

#### 列表字段

| 字段 | 说明 |
|---|---|
| 网段名称 | 用户填写，例如“生产网”“设备网” |
| 所属节点 | 上报该网段的客户端 |
| 真实网段 | 例如 `192.168.1.0/24` |
| 映射网段 | 例如 `100.64.1.0/24` |
| 状态 | 待确认 / 已启用 / 已停用 / 异常 |
| 是否冲突 | 是否存在相同真实网段 |
| 备注 | 用户或管理员备注 |

#### 网段状态

```text
待确认：客户端上报后等待管理员确认
已启用：已分配映射网段并参与路由
已停用：不再下发给实施端或其他客户端
异常：客户端离线、路由失败或映射冲突
```

#### 操作

```text
确认启用
停用网段
修改名称
修改备注
重新分配映射网段
查看访问权限
查看访问测试
```

重新分配映射网段属于高影响操作，界面需要提示已有访问地址会变化。

### 映射关系

#### 页面目标

清晰展示真实网段和平台映射网段的对照关系。

#### 列表字段

| 字段 | 示例 |
|---|---|
| 客户/节点 | 客户 A |
| 网段名称 | 生产网 |
| 真实网段 | `192.168.1.0/24` |
| 映射网段 | `100.64.1.0/24` |
| 访问示例 | `192.168.1.10 -> 100.64.1.10` |
| 状态 | 已启用 |

#### 展示要求

映射关系页需要支持复制：

```text
复制真实网段
复制映射网段
复制访问示例
复制完整映射说明
```

完整映射说明示例：

```text
客户 A / 生产网：真实网段 192.168.1.0/24，映射网段 100.64.1.0/24。访问真实设备 192.168.1.10 时，请使用 100.64.1.10。
```

### 权限配置

#### 页面目标

配置普通客户端和实施端能访问哪些网段。

#### 权限对象

```text
普通客户端
实施端
节点组
```

初版可以先做节点级权限，后续再做节点组。

#### 权限模式

| 权限模式 | 说明 |
|---|---|
| 禁止访问其他网段 | 普通客户端默认状态 |
| 允许访问指定网段 | 管理员选择可访问网段 |
| 允许访问全部网段 | 实施端默认状态 |

#### 权限配置页结构

```text
左侧：访问者节点列表
右侧：权限配置面板

权限配置面板：
访问模式
可访问网段列表
已选择网段
保存
取消
```

#### 权限保存后的反馈

保存后界面需要显示：

```text
权限已保存
预计下发节点数
已下发成功节点数
下发失败节点数
```

权限修改后，如果在线节点需要重新下发路由，界面应提示“路由配置正在更新”。

### 地址池配置

#### 页面目标

配置客户映射网段池和实施端地址池。

#### 配置项

| 配置项 | 默认值 | 说明 |
|---|---|---|
| 客户映射地址池 | `100.64.0.0/10` | 用于给客户真实网段分配映射网段 |
| 单个映射网段大小 | `/24` | 初版固定为 `/24` |
| 实施端地址池 | `100.127.0.0/24` | 用于给实施端分配虚拟 IP |
| 服务端虚拟地址 | `100.127.0.1` | 服务端在实施端网络中的地址 |

#### 操作规则

```text
1. 地址池已使用后不建议修改。
2. 修改地址池需要检查是否和已有映射冲突。
3. 修改实施端地址池需要提示在线实施端重连。
4. 初版不允许不同客户共享同一映射网段。
```

### 连接会话

#### 页面目标

查看当前正在发生的连接和转发情况。

#### 列表字段

| 字段 | 说明 |
|---|---|
| 来源节点 | 发起访问的实施端或客户端 |
| 目标网段 | 被访问的映射网段 |
| 协议 | ICMP / TCP / UDP |
| 目标地址 | 映射 IP |
| 所属客户 | 目标网段所属客户端 |
| 开始时间 | 首次转发时间 |
| 最近活动 | 最近一次数据时间 |
| 状态 | 活跃 / 超时 / 结束 |

UDP 会话需要显示超时时间，避免管理员误以为 UDP 存在长连接。

### 日志诊断

#### 页面目标

帮助排查无法访问、路由失败、节点离线等问题。

#### 日志分类

```text
系统日志
节点连接日志
网段上报日志
路由下发日志
权限变更日志
转发异常日志
```

#### 诊断工具

```text
测试节点连通性
测试映射网段路由
测试 ICMP
测试 TCP 端口
测试 UDP 端口
查看节点最近心跳
查看路由下发结果
```

### 本机网段页

#### 页面目标

展示系统识别到的本机网卡和网段，供用户选择哪些网段要上报给服务端。

#### 列表字段

| 字段 | 说明 |
|---|---|
| 网卡名称 | 系统网卡名称 |
| 网卡类型 | 物理网卡 / Wi-Fi / VMware / Hyper-V / Docker / VPN / Loopback |
| IP 地址 | 本机 IP |
| 推测网段 | 例如 `192.168.1.0/24` |
| 网关 | 网卡网关 |
| 是否推荐 | 系统判断是否建议上报 |
| 状态 | 可选择 / 不建议 / 不支持 |

#### 推荐规则

```text

3. Loopback 网卡不允许选择。
4. 已存在冲突时仍允许上报，由服务端分配不同映射网段。
```

## 模块之间的界面关系

### 服务端与普通客户端

```text
普通客户端选择本机网段
        ↓
普通客户端上报服务端
        ↓
服务端生成映射网段
        ↓
服务端后台显示待确认或已启用
        ↓
普通客户端显示映射结果
```

界面联动要求：

```text
1. 客户端上报后，本地显示“等待服务端确认”。
2. 服务端启用后，客户端显示“已启用”和映射网段。
3. 服务端停用后，客户端显示“已停用”。
4. 服务端重新分配映射网段后，客户端必须同步更新。
```

### 服务端与实施端

```text
实施端连接服务端
        ↓
服务端识别实施端角色
        ↓
服务端分配实施端虚拟 IP
        ↓
服务端下发可访问网段
        ↓
实施端添加本机路由
        ↓
实施端展示可访问网段列表
```

界面联动要求：

```text
1. 实施端连接后显示虚拟 IP。
2. 服务端权限变化后，实施端网段列表自动刷新。
3. 网段所属客户端离线后，实施端显示该网段不可访问。
4. 路由添加失败时，实施端必须显示异常原因。
```

### 普通客户端与实施端

普通客户端和实施端不直接进行业务配置交互。它们之间的关系通过服务端管理：

```text
普通客户端提供网段
服务端管理映射和权限
实施端访问映射网段
```

界面上不应让普通客户端直接管理实施端权限。

## 关键弹窗

### 管理员权限提示

Windows 桌面端需要在创建虚拟网卡、添加路由时提示：

```text
当前操作需要管理员权限。
软件需要创建虚拟网卡并添加系统路由，用于访问远端映射网段。
```

操作：

```text
以管理员权限重启
取消
```

### 路由清理提示

退出客户端或实施端时提示：

```text
是否清理本次添加的系统路由？
建议清理，以避免影响本机网络。
```

操作：

```text
清理并退出
保留路由
取消
```

默认选项应为“清理并退出”。

### 映射网段变更提示

服务端重新分配映射网段时提示：

```text
映射网段变更后，实施人员原先使用的访问 IP 会失效。
请确认是否继续。
```

操作：

```text
确认变更
取消
```

### 权限变更提示

后台修改权限后提示：

```text
权限保存后会重新下发路由配置，在线节点可能短暂刷新网络状态。
```

操作：

```text
保存并下发
取消
```

## 表单字段规范

### 节点名称

```text
必填
建议长度：2 到 40 个字符
示例：客户A-现场电脑01、实施-PC-张三
```

### 网段名称

```text
必填
建议长度：2 到 40 个字符
示例：生产网、设备网、VMware 子网
```

### 真实网段

```text
必填
格式：CIDR
初版只支持 /24
示例：192.168.1.0/24
```

### 映射网段

```text
由服务端分配
客户端不可手动修改
管理员可在后台重新分配
```

### 备注

```text
选填
建议长度：不超过 200 个字符
用于记录客户现场、设备类型、用途等信息
```

## 错误提示规范

错误提示要给出原因和下一步检查方向。

### 服务端连接失败

```text
无法连接服务端。
请检查服务端地址、端口、防火墙和网络连通性。
```

### 虚拟网卡创建失败

```text
虚拟网卡创建失败。
请确认软件具有管理员权限，并检查虚拟网卡驱动是否安装正常。
```

### 路由添加失败

```text
路由添加失败。
请检查系统权限，或尝试以管理员身份重新启动软件。
```

### 无访问权限

```text
当前节点没有访问该网段的权限。
请在服务端后台为该节点授权。
```

### 目标客户端离线

```text
目标网段所属客户端已离线。
请等待客户端重新上线后再访问。
```

### UDP 超时

```text
UDP 请求未收到响应。
可能是目标设备未响应、端口未开放、网络丢包或返回路径异常。
```

|---|---|---|
| P0 | 服务端连接页 | 没有连接无法组网 |

| P0 | 权限配置页 | 普通客户端隔离依赖此功能 |

## 最小可用界面

如果要先做最小版本，建议只实现以下页面：

```text
桌面端：
1. 启动页
2. 服务端连接页
3. 客户端本机网段页
4. 客户端已上报网段页
5. 实施端可访问网段页
6. 实施端访问测试页
7. 日志诊断页

服务端 Web 后台：
1. 节点管理
2. 网段管理
3. 映射关系
4. 权限配置
5. 地址池配置
```

这套最小界面已经可以支撑核心流程：

```text
客户端上报网段
服务端分配映射
后台配置权限
实施端查看映射并访问
访问失败时做诊断
```

## 界面验收标准

界面完成后，需要满足以下标准：

```text
1. 用户能明确知道当前角色。
2. 用户能明确知道是否已连接服务端。
3. 用户能明确知道虚拟网卡和路由是否生效。
4. 客户端能选择并上报本地网段。
5. 服务端能看到真实网段和映射网段。
6. 实施端能看到可访问网段。
7. 实施端能完成真实 IP 和映射 IP 的换算。
8. 普通客户端默认不可访问其他网段。
9. 后台能为普通客户端授权指定网段。
10. 访问失败时能看到明确错误原因。
11. 退出时能清理路由。
12. Windows 和 Linux 桌面端概念一致。
```
